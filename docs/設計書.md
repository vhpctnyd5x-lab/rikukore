# 陸これ（仮）— 陸版艦これ 設計書

## コンセプト
陸上自衛隊の装備（戦車・装甲車・火砲・ヘリ等）を擬人化した育成＆戦闘ブラウザゲーム。
艦これの「陸上版」。参考文献は **「りっくじあーす」**（陸自装備擬人化作品）。

## 世界観
- プレイヤーは「駐屯地司令」として、擬人化装備＝**「隊員（りっくむすめ）」** を指揮する。
- 資源を管理し、隊員を育成・編成し、演習場や災派・防衛任務に出撃させる。

## 中核システム（艦これ対応）
| 艦これ | 陸これ |
|--------|--------|
| 母港 | 駐屯地 |
| 艦娘 | 隊員（りっくむすめ） |
| 編成 | 編成（小隊） |
| 出撃・海域 | 出撃・演習場 |
| 建造 | 配備（開発） |
| 改修・改造 | 整備・改修 |
| 資源（燃料/弾薬/鋼材/ボーキ） | 資源（燃料/弾薬/鋼材/部品） |

## 資源
- **燃料**: 出撃・移動に消費
- **弾薬**: 戦闘で消費
- **鋼材**: 配備・整備で消費
- **部品**: 配備・改修で消費

## ステータス
- HP（耐久）/ 火力 / 装甲 / 機動 / 索敵 / 射程 / 運

## 兵科（クラス）
- MBT（主力戦車）/ 機動戦闘車 / 装甲戦闘車 / 自走砲 / 対空 / 偵察 / 工兵 / ヘリ

## 開発フェーズ
1. **フェーズ1（現状）**: 駐屯地UI・編成・配備・簡易戦闘のプロトタイプ（単一HTML、localStorage保存）
2. フェーズ2: 演習場マップ・複数戦・ドロップ
3. フェーズ3: 整備/改修・任務（デイリー）
4. フェーズ4: ChatGPT生成ビジュアル組み込み（assets/characters/）

## ファイル構成
```
RikuKanColle/
├── src/        ゲーム本体（index.html / style.css / game.js）
├── data/       characters.json（隊員データ）
├── docs/       設計書
├── assets/     ChatGPT生成画像（後で差し込み）
└── backups/    作業バックアップ（タイムスタンプ付き）
```

## ビジュアル方針
画像はユーザーがChatGPTで生成し `assets/characters/{id}.png` に配置。
コードは画像が無い場合プレースホルダー（兵科アイコン）を表示する。
